當使用者要求逐一 modality 走「統計方法 walkthrough」（例如「再來是 X 的部份」「開始 walkthrough Y」），**只講 4 件事**：

1. **特徵組成**（輸入維度、如何構成、sign convention）
2. **測試類型**（test 名稱、配對 vs 獨立、permutation 設定）
3. **effect size 計算公式**（Cohen's d / Mahalanobis D² / R² etc.）
4. **計算範例**（一個 cell 的 step-by-step 數學流程）

**不要寫**：結果觀察、新發現、方向 interpretation、paper narrative、生物學解讀、比較不同 cell 的 trend。

**Why:** 使用者明確表示 "你講了很多，但我要的只有統計方法，沒有觀察" (2026-04-22 session)。方法學清點本身就是目的，解讀是另一個階段。

**How to apply**: 遇到「modality × 方法」的走讀請求，壓縮到 ≤ 一半現有長度；答案以 "特徵 → test → effect size → 範例" 4 段為限。若使用者後續要觀察再另外問。
